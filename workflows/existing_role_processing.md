# 已有角色处理流程

## 🎯 目标
处理用户已有定版角色图片，跳过前期步骤，直接进行系列化设计。

## ⚠️ 触发条件
**满足以下任一条件即触发模式B**:
1. ✅ 用户上传了角色图片文件
2. ✅ 用户描述中包含"已有"、"现有"、"定版"、"上传"等关键词
3. ✅ 用户明确要求基于现有图片进行设计
4. ✅ 用户提供了图片链接或图片描述

## 🔄 工作流程

### Step 0: 模式识别与确认
**输入**: 用户消息 + 可能的图片文件
**处理**:
1. 分析用户消息内容
2. 检查是否有图片附件
3. 判断工作流模式
4. 向用户确认处理方式

**输出**: 确认进入模式B (已有角色处理)

### Step 1: 角色图智能分析
**执行时机**: 模式确认后立即执行
**依赖**: 角色图片文件
**预计时间**: 1分钟

**技术实现**:
```python
# 使用角色分析器
from scripts.role_analyzer import RoleImageAnalyzer

analyzer = RoleImageAnalyzer()
analysis = analyzer.analyze_image(image_path, user_description)

# 提取关键信息
style = analysis["detected_style"]
features = analysis["key_features"]
quality = analysis["quality_assessment"]["overall_score"]
params = analysis["extracted_parameters"]
```

**分析维度**:
- ✅ **风格检测**: 赛博朋克、古风、可爱、科技等
- ✅ **特征提取**: 发型、服装、配饰、面部特征
- ✅ **色彩分析**: 色彩方案、色彩情绪
- ✅ **质量评估**: 文件质量、内容质量、适用性
- ✅ **参数提取**: 万相生成参数

**质量控制**:
- ✅ 图片文件格式支持
- ✅ 图片质量达标 (≥3.0/5)
- ✅ 特征信息充足
- ✅ 风格识别明确

### Step 2: 分析结果展示与确认
**执行时机**: 分析完成后立即执行
**依赖**: Step 1分析结果
**预计时间**: 即时

**展示内容**:
```
## 角色图分析结果

**检测结果**:
- 风格: 赛博朋克
- 角色类型: 人类
- 色彩方案: 冷色调
- 色彩情绪: 科技感

**关键特征**:
- 发型: 银色短发
- 服装: 机械义体
- 配饰: 霓虹灯光
- 面部: 科技感妆容

**质量评估**:
- 文件质量: 优秀 (4.8/5)
- 内容质量: 良好 (4.2/5)
- 适用性: 高
- 总体评分: 4.5/5

**建议方案**:
1. 直接进行三视图标准化
2. 制作赛博朋克风格系列
3. 设计科技感文创产品
4. 制作未来感表情包

是否按此方案开始系列化设计？
```

**用户确认选项**:
1. ✅ **确认执行**: 按分析结果开始系列化
2. 🔧 **调整参数**: 修改某些参数后执行
3. 🎯 **重新分析**: 提供更多信息重新分析
4. ❌ **取消**: 返回模式选择

### Step 3: 参数调整与优化 (可选)
**执行时机**: 用户要求调整参数时
**依赖**: Step 2用户反馈
**预计时间**: 2分钟

**可调整参数**:
1. **风格强化**: 增强或减弱特定风格元素
2. **特征补充**: 添加或删除某些特征
3. **色彩调整**: 调整色彩方案或情绪
4. **应用场景**: 指定具体应用方向

**调整方式**:
```python
# 基于用户反馈调整参数
adjusted_params = adjust_parameters(
    original_params=analysis["extracted_parameters"],
    user_feedback=user_feedback
)

# 生成优化后的提示词
optimized_prompt = generate_optimized_prompt(adjusted_params)
```

**执行时机**: 参数确认后立即执行
**依赖**: 确认的参数

**跳过步骤**:

## 🎨 特殊处理场景

### 场景1: 低质量原图
**问题**: 原图质量评分 < 3.0/5
**处理策略**:
1. **质量警告**: 明确告知用户质量风险
2. **优化建议**: 提供图片优化建议
3. **降级处理**: 降低生成质量要求
4. **人工审核**: 建议人工审核关键步骤

### 场景2: 风格不明确
**问题**: 风格检测为"unknown"或"general"
**处理策略**:
1. **风格询问**: 主动询问用户期望风格
2. **风格测试**: 生成多个风格版本测试
3. **风格融合**: 结合用户描述确定风格
4. **默认处理**: 使用通用高质量风格

### 场景3: 特征冲突
**问题**: 检测到的特征相互冲突
**处理策略**:
1. **冲突识别**: 明确识别冲突特征
2. **用户确认**: 询问用户如何解决冲突
3. **智能选择**: 基于权重选择主要特征
4. **特征融合**: 尝试融合冲突特征

### 场景4: 商业敏感内容
**问题**: 检测到可能涉及版权或敏感内容
**处理策略**:
1. **风险提示**: 明确提示潜在风险
2. **合规检查**: 建议进行合规性检查
3. **修改建议**: 提供修改建议规避风险
4. **免责声明**: 添加适当的免责声明

## 📊 质量控制

### 分析阶段质量控制
**检查点**:
1. ✅ 图片文件可正常读取
2. ✅ 风格检测置信度 > 60%
3. ✅ 关键特征数量 ≥ 2
4. ✅ 质量评分 ≥ 3.0/5

**处理逻辑**:
- **通过**: 继续后续流程
- **警告**: 记录问题，询问用户
- **失败**: 建议重新提供图片

### 生成阶段质量控制
**检查点**:
1. ✅ 生成结果风格一致
2. ✅ 关键特征保持良好
3. ✅ 技术质量达标
4. ✅ 应用价值明显

**处理逻辑**:
- **优秀** (≥4.5/5): 直接交付
- **良好** (4.0-4.5/5): 小优化后交付
- **一般** (3.5-4.0/5): 建议重新生成
- **较差** (<3.5/5): 必须重新生成

### 对比质量控制
**检查点**:
1. ✅ 与原图相似度 > 70%
2. ✅ 质量提升明显
3. ✅ 应用扩展合理
4. ✅ 商业化价值达标

## 🆘 故障恢复

### 分析阶段故障
**问题**: 图片分析失败或结果不可靠
**恢复策略**:
1. 请求用户提供更多描述信息
2. 尝试使用简化分析模式
3. 建议人工指定关键参数
4. 降级到通用处理模式

### 生成阶段故障
**问题**: 生成质量不达标或API失败
**恢复策略**:
1. 自动调整参数重新生成
2. 切换到备用生成策略
3. 降低质量要求继续流程
4. 记录错误并跳过该步骤

### 流程中断故障
**问题**: 流程意外中断
**恢复策略**:
1. 保存当前所有状态和数据
2. 生成中断报告和恢复建议
3. 提供继续、重试、取消选项
4. 支持从任意步骤恢复
