# 创意输入工作流程

## 🎯 目标
捕获用户的IP创意核心概念，通过智能提问澄清需求细节。

## 🔄 流程步骤

### Step 1: 接收创意输入
**输入方式**: 用户自由描述IP创意想法
**示例输入**:
- "我想做一个赛博朋克风格的猫娘IP"
- "需要设计一个品牌吉祥物，用于产品包装"
- "创作一个关于未来垃圾分类的科普IP"

**处理要点**:
1. 完整记录用户原始描述
2. 不打断用户的创意表达
3. 鼓励详细描述

### Step 2: 智能分析创意
**分析维度**:
1. **创意类型检测**: 角色IP、品牌IP、内容IP、文创IP
2. **关键主题提取**: 赛博、古风、可爱、科技等
3. **详细度评估**: 简略、一般、详细、非常详细
4. **缺失信息识别**: 需要澄清的关键点

**分析工具**: `scripts/interactive_questionnaire.py`

### Step 3: 生成智能提问
**提问策略**:
1. **按优先级排序**: 基于创意类型确定问题优先级
2. **渐进式提问**: 从宏观概念到微观细节
3. **示例引导**: 提供参考案例帮助用户理解
4. **追问机制**: 对模糊回答进行追问澄清

**提问维度**:
1. **创意概念** (最高优先级)
   - IP的核心创意和世界观
   - 独特之处和创新点
   - 背景设定和故事框架

2. **目标用户** (高优先级)
   - 受众群体特征
   - 用户画像和使用场景
   - 用户需求和痛点

3. **风格调性** (高优先级)
   - 视觉风格和情感基调
   - 色彩偏好和视觉元素
   - 参考案例和灵感来源

4. **应用场景** (中优先级)
   - 主要使用平台和场景
   - 输出形式和要求
   - 技术限制和兼容性

5. **商业化** (中优先级)
   - 变现路径和商业模式
   - 扩展性和可持续性
   - 版权和授权考虑

### Step 4: 交互式确认
**确认方式**:
1. **逐一提问**: 每次提出1-2个关键问题
2. **等待回答**: 给用户充分时间思考和回答
3. **即时反馈**: 对用户回答进行确认和总结
4. **灵活调整**: 根据用户回答动态调整后续问题

**确认循环**:
```
提出问题 → 等待回答 → 确认理解 → 记录信息 → 调整后续问题
```

### Step 5: 需求摘要生成
**摘要内容**:
1. **确认信息**: 用户明确确认的需求细节
2. **生成参数**: 提取的AI生成参数
3. **缺失信息**: 需要进一步澄清的点
4. **矛盾点**: 需求中的潜在矛盾

**摘要格式**:
```json
{
  "confirmed_categories": {
    "创意概念": [
      {
        "question": "IP的核心创意是什么？",
        "answer": "一个生活在赛博都市的猫娘程序员"
      }
    ]
  },
  "generation_parameters": {
    "style": "cyberpunk",
    "color_scheme": "cool_tone"
  },
  "missing_categories": ["商业化"],
  "specific_details": {
    "detail_0": "希望有机械义体和数据线元素"
  }
}
```

### Step 6: 质量控制
**质量检查点**:
1. ✅ **完整性**: 关键需求维度是否覆盖
2. ✅ **一致性**: 需求之间是否存在矛盾
3. ✅ **可行性**: 需求是否技术上可实现
4. ✅ **明确性**: 需求描述是否清晰明确

**质量阈值**:
- **必须满足**: 创意概念、风格调性
- **建议满足**: 目标用户、应用场景
- **可选满足**: 商业化细节

### Step 7: 输出到下一阶段
**输出内容**:
1. **完整需求摘要** (JSON格式)
2. **生成参数配置** (用于角色图生成)
3. **质量控制报告** (通过/警告/失败)
4. **用户确认状态** (已确认/待确认)

**交接标准**:
- ✅ 创意概念和风格调性已确认
- ✅ 生成参数已提取
- ✅ 质量控制通过
- ✅ 用户对需求摘要无异议

## 🎨 最佳实践

### 提问技巧:
1. **开放式提问**: 使用"如何"、"什么"、"为什么"开头
2. **具体化提问**: 避免模糊问题，提供具体选项
3. **示例引导**: 提供2-3个参考案例
4. **渐进深入**: 从宏观到微观，逐步深入

### 交互技巧:
1. **积极倾听**: 完整记录用户回答，不打断
2. **确认理解**: 复述用户回答，确认理解正确
3. **鼓励详细**: 鼓励用户提供更多细节
4. **灵活调整**: 根据用户反馈调整提问策略

### 错误处理:
1. **模糊回答**: 使用追问机制澄清
2. **矛盾需求**: 指出矛盾点，请求澄清
3. **技术限制**: 提前说明技术限制
4. **用户困惑**: 提供更简单的解释或示例

## 📋 成功标准

### 定量标准:
1. ✅ 至少3个需求维度得到确认
2. ✅ 生成参数提取完整
3. ✅ 用户回答平均长度 > 20字
4. ✅ 提问-回答循环 < 3轮

### 定性标准:
1. ✅ 用户理解提问意图
2. ✅ 需求描述清晰明确
3. ✅ 无重大矛盾或模糊点
4. ✅ 用户对流程满意

## 🚀 快速参考

### 常见创意类型处理:
- **角色IP**: 重点确认角色设定、性格、背景
- **品牌IP**: 重点确认品牌价值观、识别元素
- **内容IP**: 重点确认内容形式、更新频率
- **文创IP**: 重点确认产品类型、定价策略

### 紧急情况处理:
- **用户不耐烦**: 简化问题，加快节奏
- **用户困惑**: 提供更简单解释或示例
- **技术问题**: 跳过技术细节，关注创意概念
- **时间有限**: 聚焦最关键问题
