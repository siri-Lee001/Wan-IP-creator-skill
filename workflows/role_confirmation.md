# 角色图确认工作流程

## 🎯 目标
基于确认的需求生成角色图变体，通过用户确认循环获得满意的角色图。

## ⚠️ 关键原则
**角色图必须用户确认**：只有用户确认满意后，才会继续后续自动化流程。

## 🔄 流程步骤

### Step 1: 准备生成参数
**输入**: 创意输入阶段的需求摘要
**处理**:
1. 解析需求摘要中的生成参数
2. 确定主要风格和次要风格
3. 提取关键元素和特征
4. 配置生成复杂度

**工具**: `scripts/role_generator.py` - `analyze_requirements()`

### Step 2: 生成3个变体
**生成策略**:
1. **变体1: 标准版** - 完全按照分析结果生成
2. **变体2: 强化版** - 强化关键元素，增加复杂度
3. **变体3: 混合版** - 混合其他风格元素，提供创新方向

**技术实现**:
- 使用不同的提示词模板
- 调整生成参数
- 控制复杂度和细节层次

**工具**: `scripts/role_generator.py` - `generate_variants()`

### Step 3: 展示变体选择
**展示内容**:
1. **变体名称**: 描述性名称（如"赛博猫娘-标准版"）
2. **风格说明**: 主要风格和变体类型
3. **关键特征**: 材质、发型、服装、色彩等
4. **预览提示词**: 简化的生成提示词
5. **视觉参考**: 如有可能，提供类似风格的参考图描述

**展示格式**:
```
## 变体1: 赛博猫娘-标准版
**风格**: 赛博朋克（标准版）
**描述**: 银色机械义体 + 蓝色数据流 + 冷色调
**关键特征**:
  - 材质: PVC材质
  - 发型: 霓虹染发长发
  - 服装: 机械装甲外套
  - 色彩: 冷色调为主，霓虹色点缀
**预览提示词**: 纯白背景前站立PVC材质Q版动漫猫娘...
```

### Step 4: 用户选择与反馈
**用户选项**:
1. **选择变体**: "我选择变体1/2/3"
2. **修改请求**: "变体2不错，但希望把发型改成短发"
3. **重新生成**: "都不满意，重新生成"
4. **混合元素**: "混合变体1的服装和变体3的色彩"

**反馈处理**:
- **选择变体**: 记录选择，进入确认循环
- **修改请求**: 提取修改要求，重新生成
- **重新生成**: 调整参数，重新生成3个变体
- **混合元素**: 提取混合要求，生成新变体

**工具**: `scripts/role_generator.py` - `process_user_feedback()`

### Step 5: 确认循环
**循环逻辑**:
```
生成变体 → 用户反馈 → 分析反馈 → 调整生成 → 重新展示 → 直到满意
```

**循环控制**:
1. **最大循环次数**: 5次（防止无限循环）
2. **满意度阈值**: 用户明确表示"满意"或"确认"
3. **退出条件**: 用户确认满意或达到最大循环次数

**循环记录**:
- 记录每次生成的变体
- 记录用户反馈
- 记录参数调整历史
- 记录满意度变化

### Step 6: 最终确认
**确认标准**:
1. ✅ 用户明确表示"满意"、"确认"、"就用这个"
2. ✅ 用户对变体无修改要求
3. ✅ 用户同意继续后续流程

**确认动作**:
1. **保存最终变体**: 包括所有生成参数和提示词
2. **生成确认报告**: 记录确认过程和结果
3. **准备后续流程**: 传递确认结果给自动化工作流
4. **通知用户**: 告知用户确认完成，开始自动化流程

### Step 7: 质量控制
**质量检查点**:
1. ✅ **一致性**: 变体是否符合需求摘要
2. ✅ **完整性**: 变体是否包含所有关键元素
3. ✅ **可行性**: 变体是否技术上可生成
4. ✅ **用户满意度**: 用户是否对变体满意

**质量报告**:
- 生成质量评分（1-5分）
- 记录质量问题
- 提供改进建议

## 🔧 技术实现

### 变体生成算法:
```python
# 生成3个变体
variants = generator.generate_variants(analysis, requirements)

# 变体类型:
# 1. standard: 标准参数，完全按照分析结果
# 2. enhanced: 强化参数，增加复杂度和细节
# 3. hybrid: 混合参数，融合其他风格元素
```

### 反馈处理算法:
```python
# 处理用户反馈
feedback_analysis = generator.process_user_feedback(variants, user_feedback)

# 反馈类型:
# - selected_variant: 用户选择的变体ID
# - modification_requests: 修改要求列表
# - regenerate_requested: 是否需要重新生成
# - mix_requested: 是否需要混合元素
```

### 参数调整策略:
1. **风格调整**: 修改主要风格或添加次要风格
2. **细节修改**: 调整具体设计元素
3. **复杂度调整**: 增加或减少细节层次
4. **混合调整**: 融合多个变体的元素

## 📋 成功标准

### 定量标准:
1. ✅ 用户确认循环 ≤ 3轮
2. ✅ 生成变体数量 ≥ 3个
3. ✅ 用户满意度评分 ≥ 4/5
4. ✅ 确认时间 ≤ 10分钟

### 定性标准:
1. ✅ 变体符合用户创意想象
2. ✅ 用户对交互过程满意
3. ✅ 变体质量达到商业化标准
4. ✅ 确认过程流畅无阻碍

## 🚨 异常处理

### 常见问题:
1. **用户无法决定**: 提供更多参考案例，简化选择
2. **变体都不满意**: 重新分析需求，调整生成策略
3. **技术要求冲突**: 说明技术限制，提供替代方案
4. **时间压力**: 加快节奏，聚焦最关键决策

### 应急方案:
- **快速确认**: 提供2个变体而不是3个
- **简化选择**: 使用更简单的描述语言
- **参考引导**: 提供更多视觉参考
- **专业建议**: 提供专业的设计建议

## 🎯 最佳实践

### 变体设计:
1. **差异化明显**: 3个变体应有明显区别
2. **覆盖全面**: 覆盖不同的风格方向
3. **质量均衡**: 所有变体质量相当
4. **创新性**: 包含创新和突破性设计

### 交互设计:
1. **清晰描述**: 使用用户易懂的语言描述
2. **视觉辅助**: 如有可能，提供视觉参考
3. **积极引导**: 帮助用户做出决策
4. **耐心等待**: 给用户充分思考时间

### 质量控制:
1. **预先生成**: 提前生成变体，减少等待时间
2. **质量检查**: 每个变体都经过质量检查
3. **用户测试**: 基于用户反馈持续优化
4. **技术验证**: 确保变体技术上可生成
