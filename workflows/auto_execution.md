# 自动执行工作流程

## 🎯 目标
在角色图确认后，自动执行后续6步流程，无需人工干预。

## ⚠️ 触发条件
**必须满足以下所有条件**:
1. ✅ 角色图已获得用户明确确认
2. ✅ 需求摘要完整且一致
3. ✅ 生成参数已提取和验证
4. ✅ 质量控制标准已配置

## 🔄 自动执行流程

### Step 0: 准备工作
**输入**: 确认的角色图数据 + 需求摘要
**处理**:
1. 验证输入数据的完整性和一致性
2. 配置工作流参数
3. 初始化质量验证系统
4. 准备输出目录结构

**输出**: 工作流配置就绪状态

### Step 1: 三视图生成 (自动)
**执行时机**: 工作流开始后立即执行
**依赖**: 无
**预计时间**: 2分钟

**技术实现**:
```python
# 使用确认的角色图作为参考
prompt = "用图中形象生成三视图，即正面视角、侧面视角、背面视角..."
# 调用AI生成API
result = generate_image(prompt, reference_image=confirmed_role_image)
```

**质量控制**:
- ✅ 视图比例准确
- ✅ 结构一致性
- ✅ 专业标准符合
- ✅ 文件格式正确

### Step 2: 风格转换 (自动)

**依赖**: Step 1
**预计时间**: 3分钟

**质量控制**:
- ✅ 风格转换准确
- ✅ 角色识别度保持
- ✅ 色彩方案协调
- ✅ 输出质量达标

## 📊 进度监控

### 实时进度报告
**报告频率**: 每完成一个步骤立即报告
**报告内容**:
1. ✅ 已完成步骤和状态
2. ⏳ 当前执行步骤和预计剩余时间
3. 📈 总体进度百分比
4. 🔧 遇到的任何问题或警告

**报告格式**:
```
[时间戳] [步骤状态] 步骤X: 步骤名称
  状态: 进行中/完成/失败
  进度: X/Y (XX%)
  预计剩余时间: X分钟
  质量评分: X.X/5
```

### 异常处理
**监控点**:
1. **API调用失败**: 自动重试3次，失败后记录并继续
2. **生成质量过低**: 触发质量验证，低于阈值时重新生成
3. **超时处理**: 单个步骤超时后跳过，记录错误
4. **资源不足**: 降低生成质量或数量，保证流程继续

**处理策略**:
- **轻微问题**: 记录警告，继续执行
- **中等问题**: 尝试修复，最多重试2次
- **严重问题**: 暂停流程，等待人工干预

## 📋 质量控制

### 自动质量验证
**验证时机**: 每个步骤完成后立即验证
**验证维度**:
1. **技术质量**: 分辨率、格式、文件完整性
2. **内容质量**: 符合需求、一致性、创意性
3. **商业质量**: 应用价值、生产可行性、市场潜力

**验证工具**: `scripts/quality_validator.py`

### 质量阈值
**通过标准**:
- ✅ 技术质量 ≥ 3.0/5
- ✅ 内容质量 ≥ 3.5/5  
- ✅ 总体评分 ≥ 3.5/5

**处理逻辑**:
- **通过**: 继续下一步骤
- **警告**: 记录问题，继续执行
- **失败**: 重新生成或跳过该步骤

### 质量报告
**报告生成时机**: 工作流完成后
**报告内容**:
1. 每个步骤的质量评分和问题
2. 工作流总体质量评估
3. 优化建议和改进方向
4. 商业化可行性分析

## 🔧 技术实现

### 核心脚本
```python
# 工作流管理器
from scripts.auto_workflow import AutoWorkflow

# 质量验证器
from scripts.quality_validator import QualityValidator

# 执行工作流
workflow = AutoWorkflow(confirmed_role_data, output_dir)
report = workflow.execute_workflow()

# 验证质量
validator = QualityValidator()
quality_report = validator.validate_workflow(workflow.results)
```

### 配置管理
**配置文件**: `config/workflow_config.json`
**配置内容**:
- 步骤参数和依赖
- 质量阈值和标准
- 输出格式和目录
- 错误处理策略

### 日志系统
**日志级别**: DEBUG, INFO, WARNING, ERROR, CRITICAL
**日志内容**:
- 步骤执行详情
- 质量验证结果
- 错误和异常信息
- 性能统计信息

**日志文件**: `logs/workflow_YYYYMMDD_HHMMSS.log`

## 🚀 性能优化

### 并行处理
**可并行步骤**:
- 风格转换的不同风格版本
- 动作延展的不同动作
- 表情包的不同表情

**并行策略**:
- 使用线程池或异步处理
- 控制并发数量避免资源竞争
- 合并相似请求减少API调用

### 缓存机制
**缓存内容**:
- 常用的提示词模板
- 风格转换参数
- 质量验证规则
- 错误处理策略

**缓存策略**:
- 内存缓存高频数据
- 文件缓存生成结果
- 数据库缓存历史记录

### 资源管理
**资源监控**:
- API调用次数和频率
- 内存和CPU使用情况
- 磁盘空间和IO性能
- 网络带宽和延迟

**优化策略**:
- 批量处理减少API调用
- 压缩图片减少存储
- 缓存中间结果减少计算
- 优先级调度优化体验

## 🆘 故障恢复

### 常见故障处理
1. **API服务不可用**:
   - 等待重试，最多3次
   - 切换到备用API服务
   - 记录错误并跳过该步骤

2. **生成质量不达标**:
   - 自动调整参数重新生成
   - 使用不同的提示词模板
   - 人工审核后决定处理方式

3. **资源耗尽**:
   - 暂停非关键步骤
   - 降低生成质量要求
   - 请求外部资源支持

4. **流程中断**:
   - 保存当前状态
   - 生成中断报告
   - 提供恢复选项

### 恢复策略
**自动恢复**:
- 重试失败步骤
- 跳过问题步骤继续
- 调整参数重新生成

**人工干预**:
- 提供详细错误报告
- 建议处理方案
- 等待用户决策

**数据恢复**:
- 定期保存进度状态
- 备份关键生成结果
- 记录所有操作日志
