# 🦊 Siri万相IP系列化设计技能 (v1.7.4 2026.04.13 稳定版)
## 📖 概述
本技能是基于阿里云万相WAN2.7双模型打造的全自动化IP系列化生成工具，专为角色IP创作、文创周边设计、表情包制作、IP产品化场景打造。支持从零创建IP、单图生成系列成果、三视图扩展等多种模式，全流程自动化执行，可快速产出完整可商用的IP系列化成果。
## ✨ 核心特性
- **多模式适配**：支持三种输入模式：文字描述从零创建、单张角色图扩展、已有三视图直接系列化
- **专业三视图生成**：2026.04.13最新升级，输出可直接导入3D软件的8K建模级参考图，严格锁定原图特征，零创作纯转换
- **九合一效率优化**：所有系列化成果均采用九宫格单张大图输出，大幅降低API调用成本，提升生成效率
- **全流程自动化**：从输入到最终HTML展示页、成果交付一站式完成，无需人工干预
- **多环境兼容**：适配Windows、企业网络环境，无第三方依赖，编码兼容性强
- **双执行策略**：支持分步确认模式（每步生成后确认再继续）和全自动模式（直接跑完所有流程）
## 🚀 支持的输入模式
### 📌 模式1：已有角色三视图输入
- **输入**：已有的角色三视图
- **输出**：8件系列化成果
- **流程**：直接进入系列化生成环节 → 风格转换 → 动作延伸 → 文创设计 → 表情包 → 动态视频 → HTML展示页
### 📌 模式2：单张角色图输入
- **输入**：单张清晰的角色正面图
- **输出**：9件系列化成果（含专业建模三视图）
- **流程**：生成8K专业三视图 → 进入系列化生成环节 → 后续步骤同上
### 📌 模式3：从零创建（仅文字描述）
- **输入**：IP角色文字描述
- **输出**：12件系列化成果
- **流程**：生成3张不同风格概念图供选择 → 用户选定后生成三视图 → 进入系列化生成环节 → 后续步骤同上

## 🎨 技术栈配置
### 图片生成
- 模型：阿里云万相 `wan2.7-image-pro`
- 分辨率：2K (1440*1440)
- 水印：无水印
- 调用方式：Wan-skills技能包异步调用
- 智能特性：开启`thinking_mode`和`prompt_extend`
### 视频生成
- 模型：阿里云万相 `wan2.7-i2v`
- 分辨率：720P（固定）
- 时长：5秒（固定）
- 水印：无水印
- 调用方式：Wan-skills技能包异步调用
### 核心工具
- 图片生成：`wan_skills_integrated.py`（基于官方Wan-skills代码）
- 视频生成：`wan_skills_video.py`（基于官方Wan-skills代码）
- 任务轮询：`check_wan_task_status.py`（异步任务状态检查）
- 图片处理：内置自动压缩为Base64 data URL
## 🔧 使用说明
### 1. 环境要求
- Python 3.8+
- 阿里云万相API访问权限（需自行配置API密钥）
- 至少2GB可用内存
### 2. 安装依赖
```bash
cd siri-ip-series-wanxiang
pip install -r requirements.txt
```
### 3. 配置API密钥
创建`.env`文件并配置：
```ini
DASHSCOPE_API_KEY=your_api_key_here
OUTPUT_DIR=./output
```
### 4. 启动技能
在OpenClaw会话中触发，技能启动时会自动询问三个核心问题：
1. 输入模式：文字描述/单角色图片/三视图？
2. 执行策略：分步确认模式/全自动模式？
3. 确认粒度：每张图确认/仅关键节点确认/不确认？
确认后生成工作计划，按计划自动执行。
## ⚠️ 使用规范
### 输入要求
- 单张角色图尽量为正面、无遮挡、背景简单，清晰度越高生成效果越好
- 三视图需为标准正交视角，特征清晰
### 生成规范
- 所有系列化成果默认使用九宫格单张大图输出，不生成单张独立小图
- 三视图生成严格遵循"零创作纯转换"原则，不修改原图任何视觉特征
- 视频生成必须使用角色正面单主体图作为输入，禁止使用多视图、组合图
### API调用规范
- 所有请求必须包含`X-DashScope-Async: enable`请求头
- 图片生成同时需要`X-DashScope-OssResourceResolve: enable`请求头
- 异步任务生成后自动轮询状态，完成后自动下载到本地

## 💡 踩坑与成功经验总结
### 🚫 踩坑经验（避坑指南）
1. **技能引用错误**：切勿引用第三方character-modeling-sheet等外部技能，所有生图/生视频操作需直接通过万相官方API完成
2. **编码兼容性问题**：所有提示词、脚本文件中禁止使用emoji和特殊Unicode字符，避免Windows GBK编码报错
3. **API参数对齐问题**：视频API参数必须100%对齐官方文档，禁止自定义参数，否则会导致调用失败
4. **网络兼容性问题**：优先使用阿里云临时URL，避免使用固定OSS URL，解决企业网络访问限制问题
5. **返回格式适配**：万相API返回格式会不定期更新，结果解析逻辑需同步适配，避免解析失败
6. **流程完整性问题**：必须包含HTML生成和ZIP打包节点，确保成果交付完整性

### ✅ 成功经验（最佳实践）

3. **九宫格输出优化**：所有系列化成果采用九宫格单张大图输出，大幅降低API调用成本，提升生成效率
4. **多环境适配**：移除所有emoji和特殊字符，完美适配Windows和企业网络环境
5. **全流程自动化**：从输入到HTML展示、ZIP打包全流程自动化，无需人工干预
