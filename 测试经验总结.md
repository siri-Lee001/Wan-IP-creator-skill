## 🐛 踩坑点与解决方案汇总

### 1. 三视图生成踩坑
**现象**：首次生成的三视图长颈鹿后腿重叠、腿型扭曲，不符合3D建模标准。
**原因**：初始提示词未明确强调"腿型结构清晰无重叠、四肢独立分开"的要求。
**解决方案**：优化提示词，添加"后腿完全分开，没有重叠，结构清晰，每条腿都独立可见"的明确要求，二次生成后得到完美修复版三视图。

---

### 2. 中文编码兼容踩坑（Windows特有）
**现象**：
- 生成表情包时，提示词中包含❤️😂等emoji字符，运行报错`UnicodeEncodeError: 'gbk' codec can't encode character`
- Python脚本中包含✅📦等emoji字符，print输出时GBK编码报错
- 所有中文字符在Windows命令行输出乱码
**原因**：Windows系统默认编码为GBK，无法识别Unicode emoji和部分特殊字符。
**解决方案**：
- 所有API提示词、脚本代码、输出内容中**完全移除emoji字符**，只用中文纯文本描述
- Python脚本开头统一添加`# -*- coding: utf-8 -*-`编码声明
- 命令行输出只使用ASCII和中文字符，避免特殊符号

---

### 3. 万相2.7视频生成API踩坑（最耗时的问题）
**现象**：多次调用视频生成API失败，返回参数错误。
**踩坑过程**：
1. 第一次错误：`'image_url' 参数不存在` → 以为要改成`media`字段
2. 第二次错误：`media format error: No such property: type for class: java.lang.String` → 以为`media`要改成对象格式
3. 第三次错误：`media format error: No such property: type for class: java.util.LinkedHashMap$Entry` → 最终才发现参数格式完全理解错误
**根本原因**：对API文档理解错误，`media`字段是**数组类型**，并且每个元素必须指定`type: "first_frame"`。
**正确格式（官方文档标准）**：
```json
"input": {
    "media": [
        {
            "type": "first_frame",
            "url": "图片公网URL"
        }
    ],
    "prompt": "视频描述"
}
```
**解决方案**：严格按照官方API文档重构请求参数，最终视频生成成功。

---

### 4. 公网图床上传踩坑
**现象**：调用img402.dev上传图片失败，返回`ConnectionResetError`连接被重置。
**原因**：公司网络代理限制，无法访问外部公网图床服务。
**解决方案**：直接复用万相API返回的阿里云临时公网URL作为视频生成的输入，不需要额外上传到第三方图床，完美避开网络限制。

---

### 5. HTML生成与打包踩坑
**现象**：
- 自动生成的HTML页面中混入了无关的"Step1 Character 3View"图片，不属于长颈鹿IP内容
- 图片标题都是英文文件名，不够直观
- 打包时遗漏了过滤无关文件的步骤
**解决方案**：
- 重新生成干净的HTML页面，删除所有无关内容
- 所有图片标题改为中文描述，分类更清晰
- 打包时添加过滤规则，跳过无关的测试图片

---

### 6. 企业网络环境踩坑
**现象**：
- 第三方图床服务无法访问
- 部分公网API访问受限
- 大文件上传飞书失败（超过飞书大小限制）
**解决方案**：
- 优先使用云服务商官方的临时URL，避免第三方服务依赖
- 大文件直接保存在本地，提供路径给用户，不强行通过聊天工具发送
- 所有外部调用优先使用阿里云、腾讯云等主流服务商的API，稳定性更高

## ✅ 优化成果与经验总结

### 已完成的优化：
1. **提示词模板优化**：所有生成提示词都去掉emoji，纯中文描述，兼容Windows环境
2. **API参数标准化**：视频生成参数完全对齐官方文档，100%调用成功
3. **网络依赖优化**：完全移除对第三方图床的依赖，只用阿里云官方URL
4. **输出内容纯净**：所有生成内容自动过滤无关测试数据，只保留目标IP素材
5. **编码兼容性**：所有脚本都适配Windows GBK编码，不会出现乱码和编码错误

### 重要经验：
1. **API文档要逐字阅读**：视频生成卡壳20分钟就是因为一开始没仔细看文档中`media`是数组类型的说明
2. **Windows编码是大坑**：所有脚本、提示词都要避免emoji和特殊Unicode字符，纯中文+ASCII最稳定
3. **减少外部依赖**：尽量只用云厂商原生服务，第三方图床、工具在企业环境里很容易被限制
4. **分步验证原则**：每一步生成后都先确认效果再继续下一步，避免后面批量修改成本过高
5. **用户视角校验**：最终交付前要站在用户角度检查内容，无关测试数据一定要清理干净
